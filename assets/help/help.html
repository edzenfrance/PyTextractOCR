Page segmentation modes

There are several ways a page of text can be analysed. The tesseract api

provides several page segmentation modes if you want to run OCR on

only a small region or in different orientations, etc.


0 Orientation and script detection (OSD) only:

This mode is used when you want to detect orientation and

script only.


1 Automatic page segmentation with OSD:

This mode is used when you have an entire page of text and

want Tesseract to analyze it fully including orientation and

script detection.


2 Automatic page segmentation, but no OSD, or OCR:

This will segment an image into regions without doing any

actual OCR.


3 Fully automatic page segmentation, but no OSD. (Default):

It's like mode 1 but without orientation or script detection.


4 Assume a single column of text of variable sizes:

Use this if your document is composed by one column

containing words in different sizes.


5 Assume a single uniform block of vertically aligned text:

Use this if your document contains just one block where all

lines are vertically aligned.


6 Assume a single uniform block of text:

Use this if your document contains just one block where all

lines have similar width/heights.


7 Treat the image as a single text line:

If you've preprocessed your data such that you're sure each

piece being processed is exactly one line long use this option.


8 Treat the image as a single word:

If you've preprocessed your data such that each piece being

processed corresponds to exactly one word use this option.


9 Treat the image as a single word in a circle:

Similar to 8 but assumes that there might be some circular

distortion on words.


10 Treat the image as a single character:

If you're preprocessing has isolated individual characters use

this option.


11 Sparse text. Find as much text as possible in no particular order:

This mode is used when you have sparse amounts of text scattered

around an image and you want to find as much of it as possible

regardless of its structure or order.


12 Sparse text with OSD:

Similar to mode 11 but also includes Orientation and Script

Detection (OSD). It's useful when the image has sparse amounts

of text scattered around it and you also need information about

script or orientation.


13 Raw line. Treat the image as a single text line, bypassing hacks that

are Tesseract-specific:

This mode treats the image as a single line of raw printed text,

bypassing internal features within Tesseract designed to handle

more complex data layouts.
